## 一、系统概述

### 1.1 系统目标

ClawDesk 是 OpenClaw 的桌面客户端，提供：
- 一键启动/管理 Gateway 服务
- 直观的任务监控和进度可视化
- 任务持久化与中断恢复
- 友好的中文界面和功能说明

### 1.2 系统边界

```
┌─────────────────────────────────────────────────────────┐
│                     ClawDesk 系统                       │
│  ┌─────────────────────────────────────────────────┐   │
│  │              前端界面层 (Vue 3)                   │   │
│  └─────────────────────────────────────────────────┘   │
│                        ↕ Tauri IPC                     │
│  ┌─────────────────────────────────────────────────┐   │
│  │              后端服务层 (Rust)                    │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
                         ↕ CLI/HTTP
┌─────────────────────────────────────────────────────────┐
│                  OpenClaw Gateway                      │
│                  (外部系统)                             │
└─────────────────────────────────────────────────────────┘
```

### 2.2 分层架构

| 层级 | 职责 | 技术实现 |
|------|------|---------|
| 视图层 | 界面渲染、用户交互 | Vue 3 + Naive UI |

| 数据层 | 本地存储、配置管理 | Tauri Store + JSON |

#### 数据结构

```typescript
// 任务状态
interface TaskState {
  taskId: string
  createdAt: string
  updatedAt: string
  status: 'pending' | 'in_progress' | 'completed' | 'failed' | 'abandoned'
  
  // 任务信息
  taskDescription: string
  userRequest: string
  
  // 进度信息
  totalSteps: number
  completedSteps: number
  currentStep: number
  
  // 检查点列表
  checkpoints: Checkpoint[]
  
  // 上下文信息
  context: {
    projectPath?: string
    techStack?: string[]
    decisions?: string[]
  }
  
  // 文件变更
  files: {
    created: string[]
    modified: string[]
    deleted: string[]
  }
}

// 检查点
interface Checkpoint {
  step: number
  title: string
  status: 'pending' | 'in_progress' | 'completed'
  completedAt?: string
  outputs: string[]
  summary?: string
}

// 任务恢复信息
interface TaskRecovery {
  hasUnfinishedTask: boolean
  task?: TaskState
}
```

## 六、接口设计

### 6.1 Tauri 命令接口

| 命令 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `start_gateway` | port, force | ServiceStatus | 启动服务 |
| `stop_gateway` | - | void | 停止服务 |
| `restart_gateway` | - | ServiceStatus | 重启服务 |
| `get_gateway_status` | - | ServiceStatus | 获取状态 |
| `get_gateway_logs` | level, limit | LogEntry[] | 获取日志 |
| `save_task_checkpoint` | taskId, checkpoint | void | 保存检查点 |
| `get_task_state` | taskId | TaskState | 获取任务状态 |
| `get_unfinished_tasks` | - | TaskState[] | 获取未完成任务 |
| `read_config` | key | any | 读取配置 |
| `write_config` | key, value | void | 写入配置 |

## 七、安全设计

### 7.1 安全策略

| 安全项 | 措施 |
|--------|------|
| 进程隔离 | Gateway 作为独立进程运行 |
| 命令白名单 | 仅允许预定义的 openclaw 命令 |
| 输入验证 | 所有用户输入进行校验 |
| 敏感信息保护 | Token、密钥加密存储 |
| 文件权限 | 仅访问允许的目录 |

## 八、性能优化

### 8.1 前端优化

| 优化项 | 措施 |
|--------|------|
| 路由懒加载 | 按需加载页面组件 |
| 组件按需加载 | Naive UI Tree Shaking |
| 虚拟滚动 | 长列表使用虚拟滚动 |
| 防抖节流 | 输入、滚动等事件优化 |
| 缓存策略 | 合理使用 Pinia 缓存 |

### 8.2 后端优化

| 优化项 | 措施 |
|--------|------|
| 异步处理 | 所有 I/O 操作异步 |
| 进程池 | 复用子进程 |
| 日志缓冲 | 批量写入日志 |
| 状态缓存 | 减少重复查询 |
