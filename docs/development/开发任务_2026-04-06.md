## 任务一：REQ-014 任务进度可视化集成

### 当前状态

| 组件 | 状态 | 说明 |
|------|------|------|
| TaskPanel.vue | ✅ 已完成 | 进度条、步骤列表 UI |
| taskProtocol.ts | ✅ 已完成 | 解析函数（parseTaskStart 等） |
| task.ts | ✅ 已完成 | 任务状态管理 |
| ChatPanel.vue | ❌ 未集成 | 未调用解析函数 |

### 待完成工作

在 `ChatPanel.vue` 的消息处理流程中集成任务进度解析：

1. **解析任务开始**
   - 当 AI 回复包含 `[TASK_START]...[TASK_END]` 时
   - 调用 `parseTaskStart()` 解析任务名称和步骤
   - 调用 `taskStore.createTask()` 创建任务
   - 调用 `taskStore.initTaskSteps()` 初始化步骤标题

2. **解析步骤完成**
   - 当 AI 回复包含 `[STEP_DONE: N]` 时
   - 调用 `parseStepDone()` 解析步骤号
   - 调用 `taskStore.completeStep()` 更新进度

3. **解析任务完成**
   - 当 AI 回复包含 `[TASK_COMPLETE]` 时
   - 调用 `taskStore.completeTask()` 标记完成

4. **解析任务失败**
   - 当 AI 回复包含 `[TASK_FAILED: 原因]` 时
   - 调用 `parseTaskFailed()` 解析失败原因
   - 调用 `taskStore.failTask()` 标记失败

### 代码位置

```
src/components/panels/ChatPanel.vue  → 修改消息处理函数
src/utils/gateway.ts                → 查看消息接收逻辑
src/stores/task.ts                  → 已有方法，直接调用
src/config/taskProtocol.ts          → 已有解析函数，直接调用
```

### 验收标准

- [x] AI 回复包含 `[STEP_DONE: N]` 时，TaskPanel 进度条和步骤状态更新
- [x] AI 回复包含 `[TASK_COMPLETE]` 时，TaskPanel 显示任务完成
- [x] AI 回复包含 `[TASK_FAILED: 原因]` 时，TaskPanel 显示失败原因
