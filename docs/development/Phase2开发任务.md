## 任务一：REQ-014 任务进度可视化

### 需求描述

用户希望看到AI执行任务的进度，知道任务是否在正常执行。

### 功能要求

1. **进度展示位置**：在 `TaskPanel.vue` 中显示任务进度
2. **显示内容**：
   - 当前任务名称
   - 进度条（如果可量化）
   - 当前步骤描述
   - 已完成步骤列表
   - 预估剩余时间（如可预估）

3. **进度显示格式**：
```
当前任务：分析项目代码结构

████████░░░░░░░░ 60%

✓ 读取项目配置文件
✓ 扫描源码目录
⟳ 正在分析依赖关系...
○ 生成架构报告
○ 输出优化建议

预计剩余: ~30秒
```

4. **状态颜色**：
   - 进行中：蓝色
   - 已完成：绿色
   - 错误：红色

### 技术方案

1. **扩展 TaskStore**：添加任务进度相关状态
   - `currentTask`: 当前任务对象
   - `taskSteps`: 步骤列表
   - `progress`: 进度百分比

2. **WebSocket 消息解析**：
   - 解析 Gateway 返回的任务状态消息
   - 更新 `TaskStore` 中的进度数据

3. **UI 组件**：
   - 进度条组件（使用 Naive UI 的 `n-progress`）
   - 步骤列表组件

### 验收标准

- [ ] 任务执行时，TaskPanel 显示进度条
- [ ] 步骤列表正确显示已完成/进行中/待完成状态
- [ ] 进度条颜色根据状态变化
- [ ] 显示预估剩余时间（如可预估）

### 参考文件

- `src/stores/task.ts` - 任务状态管理
- `src/components/panels/TaskPanel.vue` - 任务面板
- `src/components/panels/ChatPanel.vue` - 聊天面板（参考消息处理逻辑）

## 任务二：REQ-015 异常状态提示

### 需求描述

用户希望在任务异常时收到提示，可以及时处理。

### 功能要求

1. **异常类型**：
   - 任务超时（超过60秒无响应）
   - 服务断开（WebSocket 断开）
   - 上下文满（使用率 > 95%）

2. **提示方式**：
   - 弹出通知（使用 Naive UI 的 `useNotification`）
   - 状态栏显示警告图标

3. **操作选项**：
   - 任务超时：「重试」「忽略」「新开会话」
   - 服务断开：「重新连接」「查看日志」
   - 上下文满：「新开会话」「压缩上下文」「忽略」

4. **提示格式**：
```
┌─────────────────────────────┐
│ ⚠️ 任务可能卡死             │
│                             │
│ 当前任务已执行超过60秒无响应 │
│                             │
│ [重试] [忽略] [新开会话]    │
└─────────────────────────────┘
```

### 技术方案

1. **任务超时检测**：
   - 在发送消息时启动计时器
   - 60秒内未收到响应触发超时
   - 收到响应后清除计时器

2. **WebSocket 断开检测**：
   - 监听 WebSocket `onclose` 事件
   - 检测到断开时弹出提示

3. **上下文满检测**：
   - 监听 `contextUsage` 状态
   - 超过95%时弹出警告

### 验收标准

- [ ] 任务超时时显示异常提示
- [ ] 服务断开时显示异常提示
- [ ] 上下文满时显示异常提示
- [ ] 提供操作按钮可执行对应操作

### 参考文件

- `src/stores/service.ts` - 服务状态管理
- `src/components/panels/ChatPanel.vue` - 消息发送逻辑
- `src/utils/api.ts` - WebSocket 连接

## 任务三：REQ-019 多会话列表

### 需求描述

用户希望看到所有会话，可以切换到不同的对话。

### 功能要求

1. **会话列表位置**：左侧栏 `AppSidebar.vue` 的「会话列表」区域

2. **显示内容**：
   - 会话名称（默认：第一个用户消息摘要）
   - 最后消息时间
   - 当前角色

3. **交互功能**：
   - 点击切换会话
   - 新建会话
   - 删除会话
   - 重命名会话（右键菜单）

4. **当前会话高亮**：
   - 当前激活的会话显示高亮背景

5. **会话持久化**：
   - 会话列表保存在本地
   - 应用重启后恢复

### 技术方案

1. **创建 SessionStore**：
   ```typescript
   interface Session {
     id: string
     name: string
     roleId: string
     createdAt: number
     updatedAt: number
     messages: Message[]
   }
   
   // 状态
   sessions: Session[]
   currentSessionId: string | null
   
   // 方法
   createSession()
   deleteSession(id)
   switchSession(id)
   renameSession(id, name)
   ```

2. **本地持久化**：
   - 使用 `localStorage` 保存会话数据
   - 或使用 Tauri 的 `fs` API 保存到文件

3. **UI 组件**：
   - 会话列表组件（使用 Naive UI 的 `n-list`）
   - 右键菜单组件（使用 Naive UI 的 `n-dropdown`）

### 验收标准

- [ ] 左侧栏显示会话列表
- [ ] 当前会话高亮显示
- [ ] 可以新建会话
- [ ] 可以删除会话
- [ ] 可以重命名会话
- [ ] 切换会话后正确加载历史消息
- [ ] 应用重启后会话列表恢复

### 参考文件

- `src/components/layout/AppSidebar.vue` - 左侧栏
- `src/stores/role.ts` - 角色状态管理（参考）
- `src/components/panels/ChatPanel.vue` - 聊天面板（消息管理）
